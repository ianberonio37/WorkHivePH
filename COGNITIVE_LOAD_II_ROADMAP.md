> **Why this arc exists (the honest framing Ian forced).** Arc K proved jobs are *completable*; UFAI proved *element floors*; Arc V (EFFORTLESS) measured *clicks/hops/density* and scored green; Arc W made it *look* layered. **Yet a real user (Pablo, Lucena supervisor) on the real front door hit two cognitive-load defects none of those caught:** (#1) the real login never resolved his hive, so the whole dashboard was the WRONG/empty view — and **every prior arc audit ran on a Tester/recipe session that masked this**, i.e. we were grading a view real users don't get; (#2) the "Today's One Thing" alert named a specific asset + action ("Donaldson Pulse-Jet PJBH: PM due — schedule the technician") but its CTA dumped him on a **generic 30-item list** with the named asset un-selected, un-scrolled, un-actioned — the alert did the thinking, then threw it away. **Reachable + completable + low-click + pretty can all be true while the platform still makes the user re-do the thinking on every hand-off.** That residual — *task continuity and real-user cognitive load* — is this arc. (Seed findings: `COGNITIVE_LOAD_FINDINGS.md` #1, #2.)

**Spine method (same as Arcs D–W):** study → **skills-first + reputable sources** (done, see §6) → lock spine → **X0 (denominator + lens probes + frozen baseline + teeth-proven gate)** → find→fix→gate→verify→ratchet → teach → persist. **Measured-% not vibes; HARD proxies GATE, SOFT signals only RANK (fed to a vision/persona judge); Ian's eye is final arbiter; every walk runs on the REAL user view.**

## ★ The three non-negotiable corrections this arc bakes in (from Ian's critique of the prior arcs)

1. **Walk the REAL user view, or the audit is invalid.** Every probe/persona walk must establish the signed-in user's true hive context the way a real user gets it (Issue #1: the front-door login itself doesn't — that's the first fix). No more grading Tester/recipe-only sessions. A walk on the wrong/empty view is a void result.
2. **HARD-gate only deterministic proxies; SOFT signals are warn-only.** The prior arcs' headline numbers were partly driven down by **recalibrating over-flagging proxies toward green** — the "false sense" Ian forbids. This arc separates, up front, what a script can *prove* from what only a human/vision-judge can *rank*, and **never auto-passes an arc by redefining a soft proxy.** (See §4.)
3. **★ "Feature pages" ALWAYS includes the LANDING PAGE — in BOTH states (Ian, 2026-06-25: "you keep on dropping my landing page").** `index.html` is the single most-trafficked surface AND it is TWO distinct surfaces in one file: **(a) the signed-OUT marketing landing** (hero + CTAs, "Join WorkHive"/"Install", the FAQ accordion, the sign-in modal — the front door, and where Issue #1 lives) and **(b) the signed-IN home dashboard** (the "Today's One Thing" hero + KPI tiles — where Issue #2 lives). Both states are first-class, permanent members of the page denominator and **must be swept every phase, never dropped.** Per the QA/Grounded-Sweep rule: change identity → **reload** → confirm the expected wrapper (`mkt-wrap` signed-out vs `ops-home` signed-in) → then measure (clearing identity without a reload leaves the old dashboard rendered and audits the wrong page). The landing is "Wave 0" — its nav/identity/sign-in is a dependency of every other page.

## The measurement model — 6 families, 14 gated types (the lens set)

Grounded in NN/g **Interaction Cost** (reading/searching/comprehension/clicking/typing/waiting/**attention-switching**/**memory-load**), Laws of UX (Hick/Fitts/Miller/Jakob/Tesler/Doherty/Goal-Gradient/Postel), Cognitive Load Theory, Refactoring UI, GDS one-thing-per-page, Krug. Each type = a **deterministic proxy that GATES** (HARD) **or** a **vision/persona cue that RANKS** (SOFT). HARD/SOFT is explicit per the over-flag guardrail.

### Family A — Continuity & Hand-off  *(the seed family — fully deterministic, highest signal, start here)*
| # | Type | Proxy (HARD unless noted) | Fix |
|---|---|---|---|
| **A1** | **Generic-landing hand-off** (the seed) | alert/tile/row copy names a specific entity (asset tag / WO# / part) but its `href`/handler resolves to a list route with no id/param selecting that record → FAIL | deep-link `…?asset=<id>&action=<verb>`; land on the record |
| **A2** | **Lost intended action** | alert copy contains an action verb (acknowledge/start/close/schedule…) but the target has no matching focusable CTA scrolled/auto-focused → FAIL | carry intent; scroll-to + focus the matching CTA on land |
| **A3** | **State/scroll/filter amnesia** | list routes don't persist filter/sort/scroll in URL/sessionStorage; Playwright back-nav probe: filter→detail→back loses state → FAIL | persist list state in URL; restore on back |
| **A4** | **Cross-step context drop** | a multi-step flow re-asks a field already known (same name in step N+1, not prefilled) → FAIL | prefill/echo known values read-only |

### Family B — Choice & Navigation  *(Hick / information scent)*
| # | Type | Proxy | Fix |
|---|---|---|---|
| **B1** | **Choice overload / no dominant CTA** | count equal-weight interactive controls in the primary zone; FAIL if no single visually-dominant primary (pairs Arc W H-lens) · **SOFT edge** | one dominant primary; demote rest to disclosure |
| **B2** | **Low information scent** | label in generic blocklist (more/details/here/view/go) OR icon-only with no `aria-label` → FAIL (HARD); "does a specific label truly predict its target" → **SOFT** | action+object labels ("Schedule PM — Donaldson PJBH") |
| **B3** | **Dead-end / no next step** | success/empty-state container has zero primary CTA → FAIL | every terminal/empty state gets a "what's next" CTA |

### Family C — Recall vs Recognition  *(memory offload)*
| # | Type | Proxy | Fix |
|---|---|---|---|
| **C1** | **Recall-dependent input** | free-text input whose label references an enumerable entity (asset/part/WO) with no adjacent picker/datalist → FAIL | searchable picker/autocomplete |
| **C2** | **Placeholder-as-label** | input has `placeholder` but no persistent `<label>`/`aria-label` → FAIL | persistent visible label; placeholder = example only |
| **C3** | **No external memory for status** | record card lacks a status/last-action/timestamp element (presence HARD); whether it's the *right* salient state → **SOFT** | show status + last action + owner on the record |

### Family D — Hierarchy & Focus  *(mostly SOFT — overlaps Arc W; warn-only)*
| # | Type | Proxy | Fix |
|---|---|---|---|
| **D1** | **Flat visual hierarchy** | HARD signal: primary CTA shares identical computed weight/size/color with ≥3 siblings → *candidate flag*; "reads as hierarchy" → **SOFT (judge)** | one dominant primary; de-emphasize the rest |
| **D2** | **No one-thing-per-page** | count independent top-level primary-task regions/forms per route; WARN >1 (esp. mid-flow) | split into single-task steps |
| **D3** | **Decorative noise over data** | decoration:data node ratio computable but threshold = **SOFT (judge)** | omit needless words/chrome; let data dominate |

### Family E — Feedback & Recovery  *(Doherty / error)*
| # | Type | Proxy | Fix |
|---|---|---|---|
| **E1** | **Unresponsive action (>400ms, no feedback)** | Playwright: ms from primary-CTA click to first visual change (spinner/disabled/optimistic); FAIL >400ms with nothing (pairs Arc V F-lens) | optimistic UI / instant busy state |
| **E2** | **Unactionable error** | validation error is generic ("Invalid"), not field-adjacent (`aria-describedby`), or wipes input → FAIL (structural HARD); wording quality → **SOFT** | field-adjacent, plain-language, input-preserving |
| **E3** | **Destructive action w/o recovery** | control with destructive verb (delete/close/cancel) lacks confirm OR undo in DOM/handler → FAIL | undo toast / confirm on destructive ops |

### Family F — Density & Reach  *(Fitts / mobile — partly covered by validate_mobile + Arc W; extend)*
| # | Type | Proxy | Fix |
|---|---|---|---|
| **F1** | **Sub-min / crowded touch targets** | bbox <44–48px or adjacent-target gap <8px at mobile vp → FAIL (extends `validate_mobile.py`) | ≥44–48px targets + ≥8px gaps |
| **F2** | **Primary CTA out of thumb reach** | at mobile vp, primary CTA not in bottom thumb band / below fold w/o autofocus → FAIL | sticky bottom primary on task pages |
| **F3** | **Inconsistent patterns (Jakob)** | same job re-implements components instead of reusing the canonical class (primary-CTA/status-badge/back-nav) → FAIL (pairs Arc W C-lens) | one reusable component per job, platform-wide |

**Crosswalk (no duplication of prior arcs):** B1/D1 ↔ Arc W H-lens (focal); F1/F3 ↔ Arc W C/I + `validate_mobile`; E1 ↔ Arc V F-lens (Doherty). Arc X **owns Families A + C + B2/B3 + E2/E3** (the un-measured task-continuity & recall axes); it *reuses* the others' gates rather than re-implementing them.

## ★ §4 Measurement honesty — HARD gates vs SOFT judge (the anti-false-sense contract)

**HARD — script can prove it → these are the failing gates** (lowest false-positive, start here): **A1, A2, A3, A4, B2(blocklist+icon), B3, C1, C2, D2(count), E1, E2(structural), E3, F1, F2, F3.** These are URL/route/param analysis, DOM presence/attribute facts, bounding-box math, and Playwright timing/back-nav probes — objective.

**SOFT — warn-only, fed to a vision/persona judge, NEVER auto-fail, Ian's eye is arbiter:** **D1 (reads-as-hierarchy), D3 (too-noisy threshold), B2 (label semantic adequacy), C3 (status salience), E2 (wording quality).**

**The rule (so we don't repeat the prior-arc mistake):** an Arc X phase reaches 100% by driving the **HARD** floors to target AND getting Ian's eye to sign off the **SOFT** ones — **never** by recalibrating a soft proxy until the number goes green. If a HARD proxy is found to over-flag a legitimate pattern, we *narrow the proxy with evidence and say so in the log* (as a calibration, dated, with the offender shown) — but we do not move a SOFT signal into the HARD gate to manufacture a pass.
