## §0 — VERIFICATION DOCTRINE (inherits NEXT_ARCS §0)

Every row is proven by a **live fault-injection walk** — deliberately break a flow and verify it is SEEN (Sentry/error-tracker) + surfaces on a dashboard (Grafana) within the SLO window — not by a static check alone.

**Specialized personas (the operators) — each walks LIVE:**
- **On-call SRE** — when something breaks, does an ACTIONABLE alert fire (right signal, severity, links to the trace)? Not a noisy false-alarm.
- **Fault-injector** — kills an edge fn / exhausts a quota / sends a bad payload → verifies it's SEEN + surfaces within the SLO window (pairs the Arc-S resilience fault harness).
- **SLO reviewer** — confirms each tier's dashboard shows the right golden signals + burn-rate.

**MCP toolkit:** Grafana MCP (dashboards, alert rules, queries) + Sentry MCP (error capture — note GlitchTip is write-mostly, [[reference_glitchtip_sentry_mcp_compat]]) + Playwright MCP (trigger the fault via the real UI) + postgres MCP (`wh_traces` / `ai_cost_log` SLI rollups).
